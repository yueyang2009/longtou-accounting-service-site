# AI 协作规则

## 内容规则

- 页面文案必须服务于企业老板的决策和预约咨询目标。
- 不夸大承诺，不写无法验证的结果。
- 正式客户案例不得暴露客户名称，除非客户已明确授权。
- 联系方式、二维码、手机号等个人信息发布前必须获得明确确认。

## 代码规则

- 复用现有组件和数据结构，避免重复造组件。
- 页面数据优先集中到 `lib/data.ts` 管理。
- 新页面使用 App Router 路由结构。
- 保持 TypeScript 类型清晰。
- 提交前运行构建检查。

## 设计规则

- 风格保持极简、克制、专业。
- 避免科技感过强的蓝色、花哨渐变和无关装饰。
- 移动端必须可读、可点、不卡布局。
- 重要转化入口保持清晰，不堆砌按钮。
